Fontys Web Carousel

Description

As I mentioned in Project 3 implementation page, I made a card/image carousel that’s a little bit different with I had in my portfolio (previously) for the third group project or Fontys’s website so I had to explicitly explain it here.

What I Did

Previously, I made a survey about the designs of our prototype and spread it among people. One of the questions asked about how I should showcase the job opportunity especially in the profile page, and majority of people answered to make it a carousel .

For this new carousel, I needed to make it moves one visible item at a time and shows multiple visible items (in this case 3 items) at once . Here’s what I understand from the previous portfolio carousel compared to Fontys website’s:

  1. Index handling:
    • First Implementation: Uses modular arithmetic to ensure the index stays within bounds (currentIndex = (currentIndex + direction + numItems) % numItems). This keeps the index within the range of [0, numItems-1].
    • Second Implementation: Adjusts the currentIndex manually and considers the total number of visible items. If the index goes below 0, it sets it to the maximum possible starting index for the last visible set of items (currentIndex = totalItems - visibleItems). For moving right, it resets to 0 if it exceeds the bounds.

  2. CSS transform calculation:
    • First Implementation: Calculates the offset based on the total width of the carousel and the current index. This means the carousel slides by the width of one item at a time.
    • Second Implementation: Calculates the offset based on a percentage of the total visible items. This allows it to slide by the width of multiple items at once if visibleItems is more than 1.

Reflection

How it went: Working on the image carousel for the Fontys website was a pretty cool experience. Before diving in, I did a survey to gather opinions on how to display job opportunities on the profile page. Most people suggested using a carousel, so I rolled with that idea.
For this new carousel, I made it so that it moves one visible item at a time and shows three items at once. Comparing it to the previous carousel in my portfolio, I noticed some key differences in how the index was handled and how the CSS transform calculations were done.

What I learned: Getting direct feedback from users via survey was super helpful. It guided the design choices and ensured the carousel met user expectations. I also learned different ways to calculate CSS transforms, which can be applied to create more dynamic and flexible carousels in the future.

What will change: I want to experiment with more advanced CSS and JavaScript techniques to make carousels even smoother and more responsive. I’ll also continue to rely on user feedback to guide design choices.
