Git

What I Did

I used Git to document my projects, whether they are group projects or individual ones. By using Git commit messages, I createdetailed history of changes made to my project, this serves as documentation for future reference , making it easier for me or others to understand the project’s evolution .

Git is also widely used skill in the software development industry. Learning Git not only improves my development practices but also my employability and career prospects .

Reflection

How it went: Using Git to document my projects has been a highly beneficial practice . It provided a systematic way to track changes , collaborate with team members, and maintain an organized history of project development. The process was initially challenging as I had to familiarize myself with various Git commands and workflows, but with consistent use, it became second nature.

what I learned: Through this experience, I learned the importance of detailed commit messages. These messages not only serve as a log of changes but also help in understanding the rationale behind each modification. This practice has significantly improved my ability to backtrack and identify specific points in the project's history . Additionally, I gained a deeper understanding of Git's branching and merging capabilities, which are crucial for managing different project features and avoiding conflicts .

What will change: Moving forward, I plan to commit more frequently to capture smaller changes . This will provide a more granular history of the project’s evolution and make it easier to identify specific changes .
